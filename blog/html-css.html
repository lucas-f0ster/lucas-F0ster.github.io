Classes vs IDs


We're writing some HTML code, making a sweet webpage like the one you're on we're having a good time, got a few snazzy DIVs, a couple IMGs and maybe a few paragraphs for flavour.

But wait! All our little pieces (or elements if you're feeling fancy) are sitting on the screen in a big vertical line, they're all plain and boring and while they have all the information, they don't look very good!
We're going to need to jazz them up a little, we're going to give them some style. 


          h1 {
            font-family: "Zen Dots";
            font-size: 30px;
          }
          h2 {
              font-family: "Roboto";
              font-weight: bold;
              font-size: 20px;
          }
          p {
              font-family: "Roboto";
              font-size: 18px;
          }

Yeah, like that!
In fact, those are some of the styles for the page you're seeing right now! 😉

If we take a look at that formatting, we can see some pieces that make sense; font-size does what it says, we can see names of fonts next to "font-family" and so on.

But what's with those little "h1" and "h2" bits? They're not in the brackets, so they must be important, right?
Well they are! Those little headings are how we reference specific elements in the HTML code (the one that builds stuff) from our CSS code (the one that tells it how to look and behave).

But what does "h1" mean?
Well, h1 is a built in part of HTML, it's the highest level (meaning biggest, most important) heading - just like the ones you can choose in Word.
There's an h1 heading at the top of this page, you're reading a "p" (or paragraph) right now.

Getting the gist? CSS applies styles to HTML elements, and they need labels so the CSS can be identified.
But what if we want to apply some CSS to something that ISN'T built into HTML? What if we want to apply a style to that snazzy block of code you just scrolled past?

Well for that, we need to describe that content, and for that we use either an ID or a Class, collectively these are known as "hooks".
So, how do we use these? What's the difference? What could they look like?

So what's the difference?


Since an ID and a Class serve basically the same function, can be swapped by changing a whole one character on the HTML and the CSS files, what's different about them? Why should you use one and not there other?

Well...

An ID is unique and a Class is not

There's your answer. An element can only have one ID, and an ID can only apply to one element on any given page, while an element can have many classes and those classes can apply to many elements over and over.

This makes an ID very useful for something that appears once on a page (like the post-container that holds everything you see) while a class is useful for applying code to an element that pops up lots of times (like the code-container holding those lovely code snippets).

Technically, CSS doesn't actually care which one you use, it can apply the same styling to an ID or a Class. But if you want to use JavaScript, that's where you'll have problems, since JavaScript really does care if you've thrown a dozen identical IDs around and will have a meltdown.
The other thing that cares? Your browser! Your browser can see each ID and can jump straight to a specific ID using a # link (like index.html#your-ID), but that's not going to work when that ID sends it to two places at once...

IDs and Classes do the same job, they're written into HTML to tell other code where things are and where to do their work.
Their difference is in how they're used, IDs are unique and for one element each while Classes aren't an can be used over and over for repeated elements.
